# HTML Basics Test (Android + Node API)

A 30-question quiz on early HTML topics (multiple choice, short text, and fill-in-the-blank) delivered as an Android app, with submissions sent to a small Node + Express server that grades each answer and stores the results.

## Repo layout

- `app/` — Kotlin + Jetpack Compose Android app.
- `server/` — Node + Express API. Stores submissions in **Postgres** when `DATABASE_URL` is set, otherwise in a local NDJSON file for development.
- `server/answer_key.json` — single source of truth for grading.
- `render.yaml` — Render Blueprint for one-click deploy.
- `server/php_deploy/` — _legacy_ PHP version of the same API, kept for reference if you ever want to deploy to a PHP host. Not used by the recommended setup.

## Recommended hosting: Render (free) + Neon Postgres (free)

This combination is free, fully HTTPS, has no anti-bot wall, and persists submissions in a real database.

### 1. Push the repo to GitHub

```powershell
cd "C:\Global\cursor projects\Apps\html-basics-android-test"
git init
git add -A
git commit -m "Initial import"
git branch -M main
git remote add origin https://github.com/<your-user>/<your-repo>.git
git push -u origin main
```

### 2. Create a free Neon Postgres database

1. Sign up at https://neon.tech (no credit card).
2. Create a project (region close to your students).
3. From the dashboard, copy the **Connection string** that looks like
   `postgresql://USER:PASSWORD@HOST/DB?sslmode=require`.

The server creates its own table on startup; no SQL setup needed. (`server/schema.sql` is included if you want to run it manually in Neon's SQL editor.)

### 3. Create a free Render web service

1. Sign up at https://render.com with your GitHub account.
2. **New + → Blueprint**, pick your GitHub repo, **Apply**.
3. Render reads `render.yaml` and creates the service. It will prompt you for:
   - **`DATABASE_URL`** — paste the Neon connection string from step 2.
   - **`HTML_TEST_ADMIN_TOKEN`** — auto-generated by Render (visible under Environment). Copy this; you'll use it to view submissions.
4. Wait for the first deploy to finish. You'll get a URL like  
   `https://html-basics-test.onrender.com`.
5. Verify:
   - `https://YOUR-URL/health` → `{"ok":true}`
   - `https://YOUR-URL/api/submissions.php?token=YOUR_ADMIN_TOKEN` → `{"ok":true,"submissions":[]}`

### 4. Point the Android app at your Render URL

Edit `app/build.gradle.kts`:

```kotlin
buildConfigField("String", "SUBMIT_BASE_URL", "\"https://YOUR-URL.onrender.com\"")
```

No trailing slash. Sync Gradle and rebuild the APK:

`Build → Build Bundle(s) / APK(s) → Build APK(s)`

APK lives at:
`app/build/outputs/apk/debug/app-debug.apk`

Install on the phone. The app posts to `/api/submissions.php` over HTTPS.

### 5. View results

Open in any browser:
`https://YOUR-URL/api/submissions.php?token=YOUR_ADMIN_TOKEN`

Each submission row includes `answers`, `graded` (per-question right/wrong), `scoreTotal`, and `scoreMax`.

### Notes on Render free tier

- Service **sleeps after ~15 minutes idle**; the first request after sleep takes ~30 s to wake up. For a classroom, hit `/health` from a browser a minute before the first student submits and you'll skip the cold start.
- 750 free instance hours per month — plenty for one always-on dev service.

## Local development

Run the server on your PC for offline testing:

```powershell
cd "C:\Global\cursor projects\Apps\html-basics-android-test\server"
npm install
npm start
```

Without `DATABASE_URL`, submissions are appended to `server/data/submissions.ndjson` and admin lookup uses the token `devtoken`.

Default URL: `http://localhost:3847`  
Health: `http://localhost:3847/health`  
List: `http://localhost:3847/api/submissions.php?token=devtoken`

### Run the Android app against the local server

For emulator: temporarily change `SUBMIT_BASE_URL` to `"http://10.0.2.2:3847"`. Then uncomment the cleartext block in `app/src/main/res/xml/network_security_config.xml` so HTTP is allowed for the emulator host alias only. Remember to revert both before building a release APK.

## Editing the quiz

- **Questions:** `app/src/main/java/com/htmlbasicstest/app/data/ExerciseBank.kt`
- **Answer key:** `server/answer_key.json` (and a copy in `server/php_deploy/` if you keep the PHP option around).

When you change either, redeploy the server and rebuild the APK so both sides stay in sync.

## Security note

Use a long random `HTML_TEST_ADMIN_TOKEN`. The token is the only thing protecting the submissions list. HTTPS is automatic on Render's URL.
